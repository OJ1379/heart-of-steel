**Hierarchical fighter state machine:**
Create a `FIGHTER_STATES` object with state handlers per RESEARCH.md Pattern 2. Each state has `enter(fighter)`, `update(fighter, input, opponent)`, `exit(fighter)` methods.

Superstate logic in the update dispatch:
- Check if airborne (`fighter.y < GROUND_Y`) -- if so, apply gravity, check landing
- Check hitstun -- if `hitstunRemaining > 0`, decrement, skip normal input, animate hitstun
- Otherwise dispatch to current state's update

State transitions return a string (new state name) or null (stay). When transition occurs, call `exit` on old state, `enter` on new state, reset `stateFrame = 0`.

**Key state implementations:**

`idle`: Advance animation frame every 15 ticks (slow idle bob). Check input: up->jumpRise, down->crouch, left/right->walkForward/walkBack (based on facing), attack edge->determine fast or power via charge detection, block->block.

`walkForward` / `walkBack`: Move `fighter.x += fighter.def.stats.walkSpeed * fighter.facing` (forward) or negative (back). Clamp to stage bounds (`STAGE_LEFT` to `STAGE_RIGHT`). Advance walk animation every 8 ticks. Return to idle if direction key released.

`jumpRise`: Set `fighter.vy = fighter.def.stats.jumpForce` on enter. Each tick: `fighter.vy += fighter.def.stats.gravity; fighter.y += fighter.vy`. Transition to jumpFall when `fighter.vy >= 0`. No air control (no horizontal input change during jump per UI-SPEC).

`jumpFall`: Continue gravity. When `fighter.y >= GROUND_Y`: snap `fighter.y = GROUND_Y`, `fighter.vy = 0`, return to idle.

`crouch`: Set `fighter.y` unchanged but note crouched state for hurtbox calculation. Animation plays once to frame 3 and holds. Return to idle when down key released.

`fastAttack`: On enter, set `attackPhase = 'startup'`, `currentAttackType = 'fast'`, `hitThisAttack = false`, increment `comboCount`. Track `stateFrame`. Phase transitions: frames 1-startup = 'startup', startup+1 to startup+active = 'active', startup+active+1 to total = 'recovery'. During recovery: check input buffer for attack press -- if `comboCount < maxChain` and attack buffered, return 'fastAttack' (chains into next hit). After total frames, reset `comboCount = 0`, return 'idle'. Map animation frames: frame 0 for startup half, frame 1 for startup end, frame 2 for active, frame 3 for recovery.

`powerAttack`: On enter, set `attackPhase = 'startup'`, `currentAttackType = 'power'`, `hitThisAttack = false`. Same phase transition logic as fast but with power frame data (12/5/18). Map animation: frames 0-2 for startup/windup (longer), frame 3 for active strike.

`block`: Set `fighter.isBlocking = true` on enter. While block key held, stay in block state, advance animation to frame 3 and hold. On block key release, `fighter.isBlocking = false`, return to idle. Per CMB-03, blocking reduces damage by 80% (handled in hit detection).

`hitstun`: Enter sets `fighter.hitstunRemaining` from the attack's hitstun value. Animate recoil. Decrement remaining each tick. When 0, return to idle.

`knockdown`: Play knockdown animation once to frame 3. Hold on frame 3 (lying on ground). After 60 frames total, return to idle (get up).

**Animation frame advancement:**
Each state handles its own frame advancement. For looping states (idle, walk), cycle `currentFrame = (currentFrame + 1) % 4` at state-specific intervals. For one-shot states (attacks, hitstun), map `stateFrame` to `currentFrame` based on frame data timing.

**Hit detection function `checkHit(attacker, defender)` (per CMB-07):**
- Return false if `attacker.attackPhase !== 'active'` (no hitbox during startup/recovery)
- Return false if `attacker.hitThisAttack === true` (prevent multi-hit per attack)
- Get attack data: `atk = attacker.def.attacks[attacker.currentAttackType]`
- Calculate hitbox in world space accounting for facing direction:
  ```
  hitbox.x = attacker.x + atk.hitbox.x * attacker.facing * attacker.def.stats.reach
  if (attacker.facing === -1) hitbox.x -= atk.hitbox.w * attacker.def.stats.reach
  hitbox.y = attacker.y + atk.hitbox.y
  hitbox.w = Math.floor(atk.hitbox.w * attacker.def.stats.reach)
  hitbox.h = atk.hitbox.h
  ```
- Calculate defender hurtbox:
  ```
  hurtH = (defender.state === 'crouch') ? defender.def.stats.crouchHurtboxHeight : defender.def.stats.hurtboxHeight
  hurtbox.x = defender.x - defender.def.stats.hurtboxWidth / 2
  hurtbox.y = defender.y - hurtH
  hurtbox.w = defender.def.stats.hurtboxWidth
  hurtbox.h = hurtH
  ```
- Return `aabbOverlap(hitbox, hurtbox)` (utility from Plan 01)

**Damage resolution function `resolveHit(attacker, defender)` (per CMB-03, CMB-06):**
Called when `checkHit` returns true. Steps:
1. Mark `attacker.hitThisAttack = true` (prevent re-hit this attack)
2. Get attack data: `atk = attacker.def.attacks[attacker.currentAttackType]`
3. Calculate base damage: `baseDamage = Math.floor(atk.damage * attacker.def.stats.power)`
4. Apply block reduction: if `defender.isBlocking`, `damage = Math.floor(baseDamage * 0.2)` (80% reduction per CMB-03). Else `damage = baseDamage`.
5. Apply damage: `defender.health = Math.max(0, defender.health - damage)`
6. Apply hit stop: if `attacker.currentAttackType === 'power'`, call `applyHitStop(5)` (per D-06 / CMB-06). Else call `applyHitStop(3)`.
7. If NOT blocking: transition defender to `'hitstun'` state, set `defender.hitstunRemaining = atk.hitstun`. Push defender back: `defender.x += 8 * attacker.facing` (knockback).
8. If blocking: small pushback `defender.x += 3 * attacker.facing`. Stay in block state.
9. Clamp defender position to stage bounds.
10. If `defender.health <= 0`: transition defender to `'knockdown'` state, set `defender.knockedDown = true`.

**Hit stop system (per CMB-06, D-06):**
- Global `hitStopFramesRemaining = 0`
- `applyHitStop(frames)`: set `hitStopFramesRemaining = frames`
- In fight scene `update()`: if `hitStopFramesRemaining > 0`, decrement and skip ALL fighter state machine updates. Still render. Still read input (responsiveness per RESEARCH.md).

**Combo chain system (per CMB-01):**
In the `fastAttack` state's update:
- During recovery phase, check the `inputBuffer` array for an attack press
- `attackBuffered` = check if any entry in the last `INPUT_BUFFER_SIZE` frames has `attack === true` AND it's a new press (edge detected)
- If `attackBuffered && fighter.comboCount < fighter.def.attacks.fast.maxChain`:
  - Return `'fastAttack'` to chain into next hit
  - The enter handler increments `comboCount`
- After total attack frames with no chain: reset `comboCount = 0`, return idle

**Charge detection for power attacks (per CMB-02):**
Integrate with the input system's `updateChargeDetection`:
- While in idle/walk states: if attack key is HELD (not just pressed), increment `attackHeldFrames`
- If `attackHeldFrames >= CHARGE_THRESHOLD` (12 frames) and key is released: transition to `powerAttack` state
- If `attackHeldFrames < CHARGE_THRESHOLD` and key is pressed (edge): transition to `fastAttack` state
- The fast-vs-power decision happens at the input level: fast attack fires on keydown (immediate feel per RESEARCH.md Pitfall 6). If the player continues holding, a flag is set. On release after 12+ frames, a queued power attack fires (interrupts or follows the fast attack). Alternative simpler approach: fast on tap (keydown followed by keyup within 12 frames), power on hold (keydown held 12+ frames, triggers power wind-up, strike on keyup). Use whichever approach produces the most responsive feel -- the key requirement is that fast attacks feel INSTANT with no perceptible delay.

**Wire collision checks into fight scene update:**
After updating both fighters' state machines (and only when hit stop is not active):
```javascript
if (checkHit(player, cpu)) resolveHit(player, cpu);
if (checkHit(cpu, player)) resolveHit(cpu, player);
```

**KO detection:**
After collision checks: if `player.health <= 0 || cpu.health <= 0`, transition to victory scene. Pass winner info to victory scene for display.

**Visual hit feedback (optional polish within scope):**
On hit, briefly flash the defender sprite white for 1 frame (draw a white rectangle over the sprite position during hit stop). This is simple to implement and makes hits feel impactful.
