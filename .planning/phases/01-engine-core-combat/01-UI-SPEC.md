# Phase 1 -- UI Design Contract

> Visual and interaction contract for the engine foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This is a Canvas 2D game rendered at 400x240 internal resolution. There is no DOM UI, no CSS framework, no component library. All visual elements are drawn with canvas fillRect/fillText calls. The "design system" is a set of pixel-level constants embedded in JavaScript.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none -- vanilla Canvas 2D API |
| Icon library | none -- all icons are procedural pixel art via fillRect |
| Font | Canvas fillText with monospace at 8px base size (pixel font fallback if fillText is not crisp enough at this size) |

## Spacing Scale

All spacing is in internal resolution pixels (not CSS pixels). Multiples of 4 at 400x240.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2px | Pixel art detail gaps, 1px outline + 1px space |
| sm | 4px | HUD element inner padding, text-to-border gaps |
| md | 8px | HUD bar spacing, element-to-element gaps |
| lg | 12px | Section spacing within HUD, name-to-healthbar gap |
| xl | 16px | HUD margin from screen edges |
| 2xl | 24px | Major layout divisions (HUD area height) |

Exceptions: At 400x240, the standard 8-point web scale is too coarse. This 2px-base scale gives sufficient granularity for pixel art layout. All values are even numbers to avoid sub-pixel issues.

---

## Typography

All text is rendered via `ctx.fillText()` on the game canvas. Sizes are in internal resolution pixels.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| HUD Numbers | 8px | normal (monospace) | 10px | Health values, timer countdown, round counter |
| HUD Labels | 8px | normal (monospace) | 10px | Player name, "TIME", "ROUND" labels |
| Title Logo | custom pixel art | n/a | n/a | "HEART OF STEEL" rendered as procedural pixel art, not fillText |
| Title Prompt | 8px | normal (monospace) | 10px | "PRESS ENTER TO START" |
| Splash Text | 16px | bold (monospace) | 20px | "K.O.", "FIGHT!", round announcements (Phase 2 but layout reserved) |

**Font rendering rule:** Use `ctx.font = '8px monospace'` with `ctx.textBaseline = 'top'`. If fillText at 8px does not produce crisp single-pixel strokes across Chrome, Firefox, and Safari, replace with a custom bitmap font (5x7 pixel grid per glyph, drawn with fillRect). This swap is a rendering-only change with no architectural impact.

**Text alignment:** HUD labels left-aligned for P1, right-aligned for CPU. Timer and round counter center-aligned at x=200.

### Accent Color Reserved For

Accent gold (`#FFCC00`) is reserved exclusively for:
- Won-round indicator pips
- Timer warning flash (alternates with red at <5 seconds)
- Title screen logo highlight
- Hit spark flash (1-frame white-yellow overlay on impact point)

### Destructive/Negative Color

Red (`#CC0000`) is reserved exclusively for:
- Low health bar state
- Timer running out warning
- K.O. text flash

| Element | X | Y | W | H | Alignment |
|---------|---|---|---|---|-----------|

| P1 health bar | 48 | 8 | 128 | 8 | left-fill |

## Interaction Contract

### Input Mapping

| Action | Player 1 Keys | Behavior |
|--------|---------------|----------|
| Move left | A, ArrowLeft | Walk back (if facing right) or forward (if facing left) |
| Move right | D, ArrowRight | Walk forward (if facing right) or back (if facing left) |
| Jump | W, ArrowUp | Parabolic arc, no air control |
| Crouch | S, ArrowDown | Shrinks hurtbox, hold to stay crouched |
| Fast attack | J, Z (tap) | Immediate on keydown, chainable 3-4 hits |
| Power attack | J, Z (hold 12+ frames, release) | Wind-up while held, strike on release |
| Block | K, X (hold) | 80% damage reduction while held |

### Visual Feedback

| Event | Visual Response | Duration |
|-------|-----------------|----------|
| Fast attack hit | 3-frame hit stop (both fighters freeze), health bar depletes | 3 frames |
| Power attack hit | 5-frame hit stop (both fighters freeze), health bar depletes | 5 frames |
| Block | No hit stop, reduced health bar depletion, defender stays in block pose | Instant |
| Health below 50% | Health bar color transitions from green to yellow | Permanent until healed or KO |
| Health below 25% | Health bar color transitions from yellow to red | Permanent until KO |
| Timer below 10s | Timer text turns red | Permanent for rest of round |
| K.O. | "K.O.!" splash text at screen center, losing fighter plays knockdown animation | 90 frames |
| Title screen idle | "PRESS ENTER TO START" blinks on/off | 30 frames on, 30 frames off |
