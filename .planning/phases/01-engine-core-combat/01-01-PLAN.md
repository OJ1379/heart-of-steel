Create index.html with the complete game engine foundation. The file structure follows the recommended architecture from RESEARCH.md:

**HTML/CSS shell:**
- DOCTYPE html with a single canvas element `id="game"`
- CSS: `body { margin: 0; background: #000000; overflow: hidden; display: flex; justify-content: center; align-items: center; height: 100vh; }`
- CSS on canvas: `image-rendering: pixelated; image-rendering: crisp-edges;`
- No external dependencies, no CDN links, no fetch calls (per ENG-01)

**Constants & Config section:**
- `GAME_W = 400`, `GAME_H = 240` (internal resolution per UI-SPEC)
- `TICK_RATE = 1/60` (60fps fixed timestep per ENG-02)
- `MAX_FRAME_TIME = 0.25` (spiral-of-death prevention)
- `INPUT_BUFFER_SIZE = 6` (per ENG-05)
- `GROUND_Y = 192` (ground line per UI-SPEC: 48px above bottom)
- `STAGE_LEFT = 24`, `STAGE_RIGHT = 376` (fight area bounds per UI-SPEC)
- `CHARGE_THRESHOLD = 12` (frames to hold for power attack per CMB-02)

**Canvas setup (per ENG-03):**
- `canvas.width = GAME_W; canvas.height = GAME_H;`
- `ctx = canvas.getContext('2d', { alpha: false });`
- `ctx.imageSmoothingEnabled = false;`
- `resizeCanvas()` function: scales canvas to fill window maintaining 5:3 aspect ratio using `Math.min(window.innerWidth / GAME_W, window.innerHeight / GAME_H)`, sets `canvas.style.width` and `canvas.style.height` with `Math.floor()`. Listens on `window 'resize'` event.

**Input system (per ENG-05):**
- `keys = {}` object tracking keydown/keyup state via `event.key`
- `window.addEventListener('keydown', e => { keys[e.key] = true; e.preventDefault(); })`
- `window.addEventListener('keyup', e => { keys[e.key] = false; })`
- `readInput()` function returns an input snapshot object: `{ left: keys['a'] || keys['ArrowLeft'], right: keys['d'] || keys['ArrowRight'], up: keys['w'] || keys['ArrowUp'], down: keys['s'] || keys['ArrowDown'], attack: keys['j'] || keys['z'] || keys['J'] || keys['Z'], block: keys['k'] || keys['x'] || keys['K'] || keys['X'], ultimate: keys['l'] || keys['c'] || keys['L'] || keys['C'], enter: keys['Enter'] }`
- `inputBuffer = []` circular buffer array, push input each tick, shift when length exceeds INPUT_BUFFER_SIZE
- `attackHeldFrames = 0` counter for charge detection per RESEARCH.md Pattern 4
- `updateChargeDetection(input)` function: increments `attackHeldFrames` while attack held, returns `'power'` on release after 12+ frames, `'fast'` on release after <12 frames, `null` while held or no input
- Separate `attackPressed` and `attackReleased` edge detection using previous frame state

**Game loop (per ENG-02):**
- `accumulator = 0`, `lastTime = 0`
- `gameLoop(timestamp)` function using `requestAnimationFrame`:
  - Calculate dt = (timestamp - lastTime) / 1000, clamp to MAX_FRAME_TIME
  - Accumulate dt, while accumulator >= TICK_RATE: call `update()`, subtract TICK_RATE
  - Call `render()` after update loop
  - `requestAnimationFrame(gameLoop)` at end
- `update()` function: calls `readInput()`, then dispatches to `scenes[currentScene].update(input)`
- `render()` function: clears canvas with `ctx.fillStyle = '#000000'; ctx.fillRect(0, 0, GAME_W, GAME_H);`, then dispatches to `scenes[currentScene].render(ctx)`

**Scene manager (per ENG-06):**
- `currentScene = 'title'` (start state)
- `scenes` object with three scene objects, each having `enter()`, `update(input)`, `render(ctx)` methods
- Scene transition: when `update` returns a non-null string, call `scenes[newScene].enter()`, set `currentScene = newScene`
- **Title scene:** Renders dark background (#0A0A1A per UI-SPEC), pixel art "HEART OF STEEL" logo centered at y=60 (built from fillRect calls, approximately 160px wide x 40px tall using blocky capital letters in white #FFFFFF with gold #FFCC00 highlights), "PRESS ENTER TO START" text at y=190 centered using `ctx.font = '8px monospace'; ctx.textAlign = 'center'; ctx.fillText(...)` that blinks on/off every 30 frames using a frame counter. Returns `'fight'` when `input.enter` is true.
- **Fight scene:** `enter()` initializes fight state (placeholder -- fighters added in Plan 02). `render(ctx)` draws the cached stage background via `ctx.drawImage(stageCanvas, 0, 0)`. `update(input)` is a placeholder that returns null (combat logic added in Plan 02).
- **Victory scene:** `render(ctx)` draws dark background, "K.O.!" text at center (16px bold monospace, white), "PRESS ENTER TO CONTINUE" blinking at y=190. `update(input)` returns `'title'` when enter pressed.

**Sprite cache infrastructure (per ENG-04):**
- `spriteCache = {}` object
- `initSpriteCache(fighterDefs)` function: iterates fighter definition array, for each fighter iterates `Object.entries(def.sprites)`, for each state maps frame draw functions to offscreen canvases created via `document.createElement('canvas')` with `width = def.spriteWidth || 64` and `height = def.spriteHeight || 56`, calls `drawFn(octx)` to paint, stores canvas reference in `spriteCache[def.name][state][frameIndex]`
- `renderFighter(ctx, fighter)` function: looks up `spriteCache[fighter.def.name][fighter.state]`, gets frame at `fighter.currentFrame % frames.length`, uses `ctx.save/restore` with `ctx.scale(-1,1)` for left-facing, draws with `ctx.drawImage()` at integer coordinates using `Math.floor()`
- Called at init with empty array for now (fighters added in Plan 02)

**Utility functions:**
- `aabbOverlap(a, b)` -- returns true if two rectangles {x, y, w, h} overlap (per RESEARCH.md code example)
- `clamp(val, min, max)` -- returns clamped value
- `drawText(ctx, text, x, y, color, size, align)` -- wraps fillText with `Math.floor` coordinates, sets font to `${size || 8}px monospace`, sets textAlign, adds 1px black shadow by drawing text at (x+1, y+1) in #000000 first then the actual text on top
- `frameCounter = 0` global, incremented each tick, used for blinking and animation timing

**Init sequence:**
- `window.addEventListener('load', () => { ... })` or immediate IIFE at script end
- Call `resizeCanvas()`
- Call `initSpriteCache([])` (empty for now)
- Call `scenes[currentScene].enter()`
- Call `requestAnimationFrame(gameLoop)`
