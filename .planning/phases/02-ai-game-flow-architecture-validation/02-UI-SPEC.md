# Phase 2 -- UI Design Contract

> Visual and interaction contract for AI, game flow, and architecture validation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This is a Canvas 2D game rendered at 400x240 internal resolution. There is no DOM UI, no CSS framework, no component library. All visual elements are drawn with canvas fillRect/fillText calls. The "design system" is a set of pixel-level constants embedded in JavaScript. Phase 1 contract remains canonical for all inherited elements (HUD, stage, Trump palette, title screen).

**Portrait rendering:** Each portrait is a simplified bust view of the fighter (head + shoulders) drawn with fillRect at 40x48 resolution on an offscreen canvas. Uses the fighter's full color palette but at portrait scale. Trump shows orange skin, gold hair, navy suit, red tie. Biden shows fair skin, silver hair, navy suit, blue tie, aviator sunglasses.

**Selection behavior:** Left/Right arrows (or A/D) move selection cursor. Selection cursor is a 2px gold border with soft glow. When player selects a fighter with Enter, CPU auto-assigns the other fighter (no mirror matches). P1 and CPU badges appear after selection.

**Difficulty selection:** Up/Down arrows (or W/S) cycle difficulty before confirming. Default is Medium. Arrow indicators ("<" and ">") bracket the current selection.

```

|  ====================================  |  y: 125

```

| Element | X | Y | W | H | Duration |
|---------|---|---|---|---|----------|
| Backdrop band (ROUND) | 0 | 85 | 400 | 50 | 60 frames |

**Timing:** Round splash plays at the start of every round (including round 1). Total splash duration: 90 frames (60 for ROUND + 30 for FIGHT). Input is blocked during splash. Fight begins immediately after FIGHT! disappears.

### Biden Visual Distinctions

Biden must be visually distinguishable from Trump at a glance at 400x240:
- **Silver hair** vs Trump's gold -- most visible differentiator at distance
- **Blue tie** vs Trump's red tie -- second strongest signal
- **Aviator sunglasses** -- signature accessory, drawn as 2px-tall dark rectangle across eyes
- **Slightly taller stance** -- Biden's sprite stands 2px taller than Trump (50px vs 48px sprite) to convey "tanky" archetype
- **Wider shoulders** -- 2px wider torso to convey bulk/reach

### Biden Stats (visual implications)

| Stat | Value | Visual Effect |
|------|-------|---------------|
| walkSpeed | slowest (~1.2) | Noticeably slower walk animation cycle |
| health | highest (~120) | Longer health bar depletion time |
| reach | longest | Attack hitboxes extend further from body |
| power | medium | Standard hit flash, standard screen shake |

## AI Behavior -- Visual Contracts

AI behavior must produce visible differences across difficulty levels:

| Difficulty | Reaction Delay | Block Rate | Visual Tells |
|------------|---------------|------------|--------------|
| Easy (250ms / 15 frames) | CPU visibly waits before acting | 20% | Frequently walks into attacks, rarely blocks, single attacks only |
| Medium (150ms / 9 frames) | CPU reacts within a beat | 50% | Blocks some attacks, uses 2-hit combos, occasional retreats |
| Hard (65ms / 4 frames) | CPU reacts almost instantly | 80% | Blocks most attacks, uses full combo chains, aggressive approach + retreat patterns |

No new visual elements needed for AI -- behavior differences are expressed through existing combat animations and timing.

## Scene Flow Contract

Phase 2 changes the scene flow from Phase 1:

**Phase 1 flow:** Title -> Fight -> Victory -> Title

**Phase 2 flow:** Title -> CharSelect -> Fight -> Victory -> Title

| Transition | Trigger | Notes |
|------------|---------|-------|
| Title -> CharSelect | ENTER key | Replace direct-to-fight from Phase 1 |
| CharSelect -> Fight | ENTER key (after fighter + difficulty selected) | Passes selected fighter def + CPU def + difficulty to fight scene |
| Fight -> Victory | ENTER key (after match ends) | Passes winner name + fighter defs |
| Victory -> Title | ENTER key (after 60 frame delay) | Resets all match state |

**Stage select (FLW-03) note:** Requirements list stage select, but Phase 2 only has 1 stage (White House). Stage select screen is deferred to Phase 3 when additional stages exist. Phase 2 auto-selects White House.

## Interaction Contract

### Character Select Screen Input

| Action | Keys | Behavior |
|--------|------|----------|
| Move cursor left | A, ArrowLeft | Move highlight to previous portrait |
| Move cursor right | D, ArrowRight | Move highlight to next portrait |
| Change difficulty | W/S, ArrowUp/ArrowDown | Cycle through Easy/Medium/Hard |
| Confirm selection | Enter | Lock in fighter choice, CPU auto-picks other, transition to fight |
| Return to title | Escape | Return to title screen (optional quality-of-life) |

**Cursor behavior:** Cursor wraps around (left from first slot goes to last available slot). Locked slots (slot 3) are skipped. Cursor movement has no repeat delay on first press; 12-frame repeat delay on held keys.

### Round Splash Interaction

| Event | Behavior |
|-------|----------|
| Round start | All input blocked for 90 frames during ROUND N + FIGHT! splash |
| Between rounds | 30-frame pause, then round splash for next round |

### Victory Screen Input

| Action | Keys | Behavior |
|--------|------|----------|
| Play again | Enter | Returns to title screen after 60-frame lockout |

### Visual Feedback (Phase 2 additions)

| Event | Visual Response | Duration |
|-------|-----------------|----------|
| Portrait selected | Gold border appears, P1 badge shows, CPU badge on other fighter | Instant |
| Difficulty changed | Selected option turns gold, others turn gray | Instant |
| Round start | "ROUND N" banner, then "FIGHT!" banner | 90 frames total |
| Match win | Victory screen with winner sprite and "WINNER!" text | Until Enter |
| Cursor move on select screen | Gold border jumps to new portrait, subtle 1-frame flash | Instant |
